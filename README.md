# Projeto Docker

Repositório com cinco desafios sobre Docker, microsserviços e comunicação entre containers. Cada pasta possui README próprio com arquitetura, execução e decisões técnicas.

- `desafio1`: comunicação cliente-servidor containerizada.
- `desafio2`: persistência com volumes compartilhados.
- `desafio3`: aplicação web containerizada servindo conteúdo estático.
- `desafio4`: dois microsserviços Flask independentes se comunicando via HTTP.
- `desafio5`: API Gateway centralizando acesso às APIs de usuários e pedidos.
